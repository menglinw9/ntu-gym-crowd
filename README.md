# ntu-gym-crowd (台大健身房人流監測)

這是一個基於 **R 語言** 與 **GitHub Actions** 建立的自動化監測系統，旨在長期追蹤國立臺灣大學體育館（新體育館）健身房的即時人流數據，以利後續進行熱點時段分析與人流預測。

## 🎯 專案目標
在使用台大新體的健身房時，往往會覺得人太多，如果想要避開人潮，最好的方法就是先了解什麼時候人會比較少？台大體育場館預約系統雖提供即時人數資訊，但缺乏歷史紀錄。本專案透過自動化爬蟲，每 15 分鐘抓取一次數據，並累積為長期的時序資料（Time-series data），幫助使用者避開擁擠時段，規劃最理想的運動時間。

- **開發語言**: `R`
- **自動化**: `GitHub Actions`

## 📊 數據說明
抓取後的資料存儲於 `ntu_gym_flow.csv`，包含以下欄位：
| 欄位名 | 說明 |
| :--- | :--- |
| `ts_site` | 系統抓取當下的台灣時間 (ISO 8601 格式) |
| `people_now` | 當前在場人數 |
| `optimal` | 場館建議最適人數 |
| `max_capacity` | 場館最大乘載量 |
| `ts_reported` | 官網顯示的最後更新時間 |
| `source_url` | 原始資料來源 |

## 🚀 執行流程
1. **Scraping**: 每 15 分鐘由 GitHub Actions 喚起 R 環境 。
2. **Cleaning**: 處理網頁中的不換行空格 (`\u00A0`) 與多餘空白。
3. **Saving**: 透過 `write_csv` 的 `append` 模式，自動判斷是否建立新檔或接續寫入數據。
